## Localization Key Generation

### Prerequisites

Before running, make sure that:
- You have `nodejs` installed. If not, you can download it [here](https://nodejs.org/en/download) (paste and run the provided script in `PowerShell`/`Bash`).
    - After installation, run the following command in the console (in the same directory as the localizer): `npm install`.
- There is a `.env` file (just ".env") in the root of this project. If it doesn't exist, create one (see `TEMPLATE.env` for an example).
- The localization system is located inside the project with the `Resources` folder (if the folder is not found nearby, the search moves up one directory level).

### Usage

To run localization generation: execute `generate-locale.bat` on Windows or `generate-locale.sh` on Linux.

If the localization has been updated by the official developers, run the generation again. The files will be updated and the previous localization of each key will be noted.

If you need to reset your localization (e.g. made many mistakes during translation), delete the file and run the generation again. Make sure to delete the file generated by this tool, not the files from the official developers.

### Details

<details>
<summary>How localization key generation works (expand)</summary>

1. Searches for file paths and determines where to place the DB (database) based on data from `.env` and `common.ts`.
2. Initializes the DB (if it didn't exist).
3. Attempts to load localization based on data from the database, if the files on the device were not found and the database is not empty.
4. Generates localization from sources (`Locale/en-US` and `Prototypes`).
    1. If a file already exists in the DB, checks whether it has changed using a hash (a hash is a number sequence generated based on the file's contents).
5. Removes garbage from the disk and DB (if files are no longer present in the sources, they are considered garbage).
    1. Removes empty directories.
</details>
