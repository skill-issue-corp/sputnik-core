## Runtime and Environment Variables

The program runs on `nodejs`, so make sure to install it [from here](https://nodejs.org/en/download) (run the listed commands in `PowerShell`/`Bash`).

Create a file named `.env` and copy the contents from `TEMPLATE.env` into it.

## Generating and Updating Localization

### Running

To start generating localization, run `generate-locale.bat` on Windows or `./generate-locale.sh` on Linux.

If the official developers release a localization update, run the generation again. The files will be updated and any previously existing localization for changed keys will be noted as TODO comments.

If you need to reset your localization (e.g. you made many translation errors), delete the file and run the generation again (delete the file generated by this tool, not the files from the official developers).

### Details

<details>
<summary>How localization key generation works (expand)</summary>

1. Locates file paths and determines where to place the database based on data from `.env` and `common.ts`.
2. Initializes the database (if it did not exist).
3. Attempts to load localization from the database if the files were not found on the device and the database is not empty.
4. Generates localization based on sources (`Locale/en-US` and `Prototypes`).
    1. If a file already exists in the database, it checks whether the file has changed using a hash (a hash is a set of numbers generated from the file's contents).
5. Removes garbage from the device and database (if files are no longer present in the sources, they are considered garbage).
    1. Removes empty folders.
</details>

## AI Locale Translation

### Setup

In the `.env` file, make sure to specify the AI model, the API endpoint URL, and the API key. This can be done for free:
- Sign up at [aistudio.google.com](https://aistudio.google.com).
- Generate a new API key and copy it (it looks like a ~40-character mix of letters and digits).
- Paste the copied key into `API_KEY` in your `.env` file (other parameters can be left as-is).

`INCLUDE_FILES` — specify paths to directories or files you want to translate. Separate multiple paths with a semicolon.

`EXCLUDE_FILES` — specify paths to directories or files you want to **exclude** from translation. Also separate multiple paths with a semicolon.

### Running

To start the AI translator, run `ai-translate.bat` on Windows or `./ai-translate.sh` on Linux.

Each iteration, the script will display the file contents, the file path, and ask for confirmation on whether to translate it. Enter "t" (for "translate") to translate the displayed file, or press Enter without typing anything to skip (you can also type anything other than "t" to skip).

### Details

<details>
<summary>How AI translation works (expand)</summary>

1. Locates file paths based on data from `.env`.
2. Collects all files and filters them according to the `INCLUDE_FILES` and `EXCLUDE_FILES` parameters.
3. Waits for translation confirmation.
4. Sends a translation request to the API with the file contents and your prompt, based on data from `.env`.
5. Updates the locale.
    1. Retrieves the translated text from the API.
    2. Generates TODO comments based on the diff between the translation and the original.
    3. Overwrites the file.
</details>
