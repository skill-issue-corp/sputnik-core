## Генерация ключей локализации

### Подготовка

Перед запуском убедитесь что:
- У вас установлен `nodejs`, если нет то можете скачать [отсюда](https://nodejs.org/en/download) (вставить и запустить указанный скрипт в `PowerShell`/`Bash`).
    - После установки выполните в консоли (по тому же пути где локализатор) команду `npm install`.
- В корне этого проекта есть файл `.env` (просто ".env"), если его нет то создайте (пример содержимого указан в `TEMPLATE.env`).
- Система локализации находится внутри проекта с папкой `Resources` (если папка не найдена рядом, поиск продолжается на уровень выше).

### Запуск

Запуск генерации локализации: запустите `generate-locale.bat` для Windows или `generate-locale.sh` для Linux.

В случае обновления локализации официальными разработчиками запустите генерацию локализации. Файлы будут обновлены и будет указана какая локализация была у ключей до этого.

В случае если нужно сбросить вашу локализацию (сделали много ошибок во время перевода) удалите файл и запустите генерацию локализации (удалите именно сгенерированный этим инструментом файл, а не файлы от официальных разработчиков).

### Детали

<details>
<summary>Как работает генерация ключей локализации (развернуть)</summary>

1. Ищет пути до файлов, определяет куда расположить БД (базу данных) на основе данных из `.env` и `common.ts`.
2. Инициализирует БД (если не существовала).
3. Пытается загрузить локализацию на основе данных из БД, если файлы на устройстве не были найдены и БД не пустая.
4. Генерирует локализацию на основе источников (`Locale/en-US` и `Prototypes`).
    1. Если файл уже есть в БД, то сверяет по хеш-коду изменился ли файл (хеш-код - набор чисел, генерируемый на основе содержимого файла).
5. Удаляет из ПК и БД мусор (если файлов нет в источниках - значит мусор).
    1. Удаляет пустые папки.
</details>
