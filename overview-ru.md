## Среда выполнения и переменные окружения

Программа работает на `nodejs`, поэтому обязательно установите его [отсюда](https://nodejs.org/en/download) (нужно выполнить указанные команды в `PowerShell`/`Bash`).

Создайте файл и назовите его `.env`, скопируйте в него содержимое из `TEMPLATE.env`.

## Генерация и обновление локализации

### Запуск

Для запуска генерации локализации - запустите `generate-locale.bat` для Windows или `./generate-locale.sh` для Linux.

В случае обновления локализации официальными разработчиками запустите генерацию локализации. 
Файлы будут обновлены и будет указана какая локализация была у ключей до этого в виде TODO комментариев.

В случае если нужно сбросить вашу локализацию (сделали много ошибок во время перевода) удалите файл и запустите генерацию локализации (удалите именно сгенерированный этим инструментом файл, а не файлы от официальных разработчиков).

### Детали

<details>
<summary>Как работает генерация ключей локализации (развернуть)</summary>

1. Ищет пути до файлов, определяет куда расположить БД (базу данных) на основе данных из `.env` и `common.ts`.
2. Инициализирует БД (если не существовала).
3. Пытается загрузить локализацию на основе данных из БД, если файлы на устройстве не были найдены и БД не пустая.
4. Генерирует локализацию на основе источников (`Locale/en-US` и `Prototypes`).
    1. Если файл уже есть в БД, то сверяет по хеш-коду изменился ли файл (хеш-код - набор чисел, генерируемый на основе содержимого файла).
5. Удаляет из ПК и БД мусор (если файлов нет в источниках - значит мусор).
    1. Удаляет пустые папки.
</details>

## ИИ перевод локали

### Подготовка

В файле `.env` обязательно укажите модель ИИ, ссылку откуда будет браться API и API ключ.
Это можно сделать бесплатно:
- Зарегистрируйтесь в https://aistudio.google.com
- Сгенерируйте новый API ключ, и скопируйте его (выглядит как набор из около 40 перемешанных цифр и букв).
- В вашем `.env` файле вставьте скопированное в `API_KEY` (остальные параметры можно не трогать).

`INCLUDE_FILES` - можно указать путь до директорий или файлов, которые хотите переводить. Для нескольких файлов/директорий указывайте через точку с запятой.

`EXCLUDE_FILES` - можно указать путь до директорий или файлов, которые **НЕ** хотите переводить (исключить). Для нескольких файлов/директорий также разделяйте точкой с запятой.

### Запуск

Для запуска ИИ переводчика - запустите `ai-translate.bat` для Windows или `./ai-translate.sh` для Linux.

Каждую итерацию скрипт будет выводить содержимое файла, путь до файла и подтверждения нужен ли перевод. 
Если хотите перевести выведенный файл введите "t" (от слова "translate"), если хотите пропустить ничего не вводите и нажимайте "Enter" (либо вводите что угодно кроме "t").

### Детали

<details>
<summary>Как работает ИИ перевод (развернуть)</summary>

1. Ищет пути до файлов на основе данных из `.env`.
2. Собирает все файлы и фильтрует их по параметрам из `INCLUDE_FILES` и `EXCLUDE_FILES`.
3. Ожидает подтверждения перевода.
4. Отправляет запрос для перевода на API с содержимым файла и вашим промтом на основе данных из `.env`.
5. Обновляет локаль.
   1. Получает переведённый текст из API.
   2. Генерирует TODO комментарии на разнице перевода и оригинала.
   3. Перезаписывает файл.
</details>
